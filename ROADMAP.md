## 建议的下一步方向

### 🔥 优先级 1 - 核心体验提升

#### 1. 离线功能
**基于 bilibili-api 的能力：**
- 视频下载与离线播放
- 专栏/动态离线保存
- 稍后再看队列离线同步

**实现要点：**
- 使用 `video.get_download_url()` 获取视频流
- 利用 Android WorkManager 后台下载
- 使用 Room 数据库管理离线内容
- 支持音频提取（podcast 模式）

#### 2. 播放体验优化
**功能：**

**基于 bilibili-api：**

## 技术栈扩展建议

### Kotlin Multiplatform (KMP) 深化
- 视频下载器 (`expect/actual`)
- 数据库层（SQLDelight）
- 网络层（Ktor 已有）

### Android 特性利用
- Jetpack Compose Material 3 深度定制
- CameraX（扫码登录）
- Biometric（生物识别锁）
- MediaSession（通知栏控制）

### bilibili-api Python 桥接
**选项 1：** Chaquopy (Python in Android)
- 直接调用 Python bilibili-api
- 完整功能，但包体积大

**选项 2：** 自行迁移 API 到 Kotlin
- 当前方案，逐步迁移
- 轻量但工作量大

**建议：** 继续 Kotlin 迁移，复杂功能（如弹幕解析）可考虑 Chaquopy

## 差异化竞争优势

### Focus 定位：
1. **纯净体验** - 无推荐算法干扰
2. **效率工具** - 快速找到想看的内容
3. **离线优先** - 适合通勤、旅行场景
4. **极简设计** - Material 3 原生体验
5. **开源透明** - 社区驱动

### 不做的事情：
- ❌ 不做推荐算法
- ❌ 不做社交功能（除基础评论）
- ❌ 不做充值/购物
- ❌ 不做复杂的社区互动
